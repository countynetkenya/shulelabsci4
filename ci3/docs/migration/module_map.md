# CI3 Module Map

## Summary

This document provides a definitive map of the legacy CodeIgniter 3 application's modules. It was generated by scanning the `mvc/` and `application/` directories to group controllers, models, views, and services by their functional domain. This map will serve as the foundation for the incremental migration to CodeIgniter 4.

## Ambiguities

*   The `Product.php.old` and `Update.php.bak` files in `mvc/controllers` and `mvc/controllers/api/v10` respectively seem to be backups and should be investigated and possibly removed.
*   The relationship between `FinanceStatement.php` and other finance modules like `Income.php`, `Expense.php`, `Invoice.php` and `Payment.php` needs to be clarified.
*   The `LedgerReconciliation` service in `application/Services/Ledger` does not have a corresponding controller. It's likely used by one of the finance modules.
*   The `DatabaseBackupService` in `application/Services/Database` is likely used by the `Backup.php` controller.
*   The `OkrCfrService` in `application/Services/Okr` is likely used by the `Cfr.php` and `Okr.php` controllers.
